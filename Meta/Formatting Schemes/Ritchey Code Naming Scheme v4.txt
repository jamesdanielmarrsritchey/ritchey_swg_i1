# Ritchey Code Naming Scheme v4

Ritchey Code Naming Scheme (RCNS) is a naming style for naming variables, and other items when coding/scripting.

RULES:
- Variables start with a prefix that consists of "var_", followed by a crc32 of a random UUID (with dashes removed), follwed by "_", followed by descriptive text, followed by "_", and ending with a shorthand for the type of variable it is. Types are as follows: "_uns" for unspecified, "_str" for string, "_arr" for arrays, "_obj" for objects, and "_num" for numbers. An example of a variable name would be "var_e16a0092_item_name_str". The same CRC32 is used across a space (a developer chosen division of code). A space will typically be a file, but the developer may choose to use smaller spaces such as per function. Per file is recommended though.
- Functions should start with a prefix that consists of "fun_", followed by a CRC32 of a random UUID (with dashes removed), follwed by "_", followed by descriptive text, followed by "_i", followed by an implementation number, followed by "_v", and ending with a version number for the function. An example of a function name would be "fun_e16a0092_captitalize_item_name_i1_v1". The implementation number is merely a count used to distinguish functions that would otherwise have the same name. For example, a function implemented in PHP, and then later implemented in Javascript, might have the same name, but the PHP version would be i1, and the Javascript version would be i2. The same CRC32 is used across a space (a developer chosen division of code). A space will typically be a file, but the developer may choose to use smaller spaces such as per function. Per file is recommended though.
- Fictional CRC32s can be used if needed.
- When dealing with items already named in a previous space, they don't need to be renamed when being used in a new space. The old name helps make it clear they are from another space, which is helpful.

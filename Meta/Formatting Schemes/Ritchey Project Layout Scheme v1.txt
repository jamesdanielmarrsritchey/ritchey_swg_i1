# Ritchey Project Layout Scheme v1

Ritchey Project Layout Scheme (RPLS) is an organizational style for source materials of projects.

RULES:
- "/": The only files here are common files such as README.md, CONTRIBUTE.md, etc. They are optional, except for README.md, which must be present, and at the very least direct users to "/Meta".
- "/Meta": This folder is for information about the project. This folder is required.
- "/Meta/Dev": This folder is for information about the project for the developers.
- "/Meta/Formatting Schemes": This folder is used to store information about formatting styles used within the project.
- "/Meta/License Texts": This folder is used to store copies of license texts for licenses used by the project.
- "/Source": This folder stores the source code, or source files of a project. This folder is required.
- "/Demo": This folder is used to store compiled binaries, or applications which demonstrate using/integrating provided source material. For example, if the source is a PHP function, there might be a script in the demo folder showing how to import, and use it.
- "/Temporary": This folder is used for any default file/folder creation by applications, or scripts, unless they are designed to use the www, or content folders for writes.
- "/Temporary 2": This folder can be used for file/folder creation by applications, or scripts, provided as resources. All writes should be done in a unique subfolder.
- "/Content": This folder is used for providing end product forms of source materials that are not code/scripts. For example if the source material is an ebook, the content folder might have it exported in a certain file format.
- "/www": This folder is used for storing websites generated by programs.
- "/Resources": This folder is for additional files that may be of use to users, or the developer.

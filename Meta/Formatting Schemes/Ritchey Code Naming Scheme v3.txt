# Ritchey Code Naming Scheme v3

Ritchey Code Naming Scheme (RCNS) is a naming style for naming variables, and other items when coding/scripting.

RULES:
- Variables should start with a prefix that consists of "var_", followed by a crc32 of a random UUID (with dashes removed), follwed by "_", followed by descriptive text, followed by "_", and ending with a shorthand for the type of variable it is. Types are as follows: "_uns" for unspecified, "_str" for string, "_arr" for arrays, "_obj" for objects, and "_num" for numbers. An example of a variable name would be "var_e16a0092_item_name_str".
- Functions should start with a prefix that consists of "fun_", followed by a CRC32 of a random UUID (with dashes removed), follwed by "_", followed by descriptive text, followed by "_i", followed by an implementation number, followed by "_v", and ending with a version number for the function. An example of a function name would be "fun_e16a0092_captitalize_item_name_i1_v1". The implementation number is merely a count used to distinguish functions that would otherwise have the same name. For example, a function implemented in PHP, and then later implemented in Javascript, might have the same name, but the PHP version would be i1, and the Javascript version would be i2.
- Fictional CRC32s can be used if needed.

# Ritchey Code Naming Scheme v1

Ritchey Code Naming Scheme (RCNS) is a naming style for naming variables, and other items when coding/scripting.

RULES:
- Variables should start with a prefix that consists of "var_", followed by a random UUID (with dashes removed), follwed by "_", followed by descriptive text, followed by "_", and ending with a shorthand for the type of variable it is. Types are as follows: "_uns" for unspecified, "_str" for string, "_arr" for arrays, "_obj" for objects, and "_num" for numbers. An example of a variable name would be "var_be19f3f56e39472ebeb2da196d17f69b_item_name_str".
- Functions should start with a prefix that consists of "fun_", followed by a random UUID (with dashes removed), follwed by "_", followed by descriptive text, followed by "_v", and ending with a version number for the function. An example of a function name would be "fun_be19f3f56e39472ebeb2da196d17f69b_captitalize_item_name_v1".
